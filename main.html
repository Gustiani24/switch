    Help

    Asset types

    • Crypto (0): Native ETH. For sell orders you must send wei equal to amount with the transaction.
    • RWA (1): Tokenized real-world assets. Use asset ID (bytes32). Fills are recorded off-chain by the escrow keeper.

    Post order

    Amount and price use 1e18 decimals. Total order value = amount × price / 1e18 (in wei). Must be ≥ min order wei. For crypto sell, send wei = amount.

    Fill order

    Send wei = fillAmount × pricePerUnit / 1e18. A fee (basis points) is sent to the treasury. Use "Fill" on a row in the order book to copy the order ID.

    Cancel

    Only the maker or the operator can cancel. For open crypto sell orders, unfilled wei is refunded to the maker.

    Contract

    Otc contract on EVM. Operator can pause, set min order size and fee bps. Escrow keeper handles RWA settlement release.

    Price decimals

    All amounts and prices use 18 decimals (1e18). Example: 1 unit = 1000000000000000000, price 2.5 = 2500000000000000000. Total value in wei = amount × price / 1e18.

    Refresh

    After posting, filling or cancelling an order, the order book and platform stats update automatically. You can also call refreshAll() in the console to refresh all data.

    Order book actions

    Click "Fill" on an open order to copy its order ID into the Fill form. Then enter the fill amount; the required wei value is shown automatically when the order is loaded.

    Transacto

    For CLI and batch operations, use the Transacto Java app with the same contract address and RPC. It can list orders, build calldata for post/fill/cancel, and export to CSV.
